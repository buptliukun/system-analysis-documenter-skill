# 系统分析文档编写技能 (system-analysis-documenter)

这个技能专门用于协助用户编写高质量的系统分析（系分）文档。通过结构化对话流程，可以引导用户完成从需求分析到文档编写的全过程。

## 核心功能
- 五步流程：模板确认 → 需求确认 → 文档撰写 → 质量评估 → 文档修订与导出
- 智能对话确认：遇到不确定的内容会主动向用户询问
- 自动质量评估：按6个维度对文档进行打分（总分100分）
- 标准化模板：提供基于DDD思想和PlantUML图表的规范系分文档结构
- 语雀导出支持：支持将生成的文档导出到语雀，充分利用PlantUML图表渲染功能

## 适用场景
- 新系统架构设计
- 现有系统重构分析
- 微服务架构设计
- 高并发系统设计
- 数据处理系统分析
- 权限管理系统设计

## 质量评估维度

文档将按照以下六个维度进行评估：
1. **完整性** (20%) - 是否覆盖所有必要内容
2. **合理性** (25%) - 技术方案和业务逻辑是否合理
3. **逻辑完整性** (15%) - 各部分之间逻辑关系是否清晰
4. **可实施性** (20%) - 技术方案是否切实可行
5. **图表质量** (10%) - 架构图和流程图质量
6. **标准化** (10%) - 符合文档编写标准

## 使用方法

在任何对话中，当需要编写系统分析文档时，直接描述你的需求，例如：
- "帮我写一个电商平台订单管理系统的系分文档"
- "我需要一个微服务架构的系统分析文档"
- "为用户权限系统重构写一份系分文档"

AI将自动启动此技能与你进行交互，最终输出一份完整的系统分析文档。

## 优势
- 结构化流程确保文档完整性
- 通过对话澄清需求避免误解
- 提供技术方案讨论和评估
- 符合行业标准的文档结构
- 支持导出到语雀以获得最佳的图表显示效果
